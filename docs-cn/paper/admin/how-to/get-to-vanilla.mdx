由于 Bukkit API 在 Paper 中的实现方式，原版和 Paper 之间的游戏体验可能存在不一致。
这可能会因为修复原版 Minecraft 中存在的错误的努力而进一步加剧。
虽然许多人可能不会注意到这些不一致，但在某些情况下它们会成为问题。
一个例子是技术玩家可能难以让他们的机器按预期工作。
本页旨在为想要尽可能接近原版的玩家提供一个起点。

:::info

本页的灵感来自 [Earthcomputer 的收集](https://gist.github.com/Earthcomputer/2296da33b8cc91dba81b48103c0e1fe3)，
其中列出了破坏原版体验的更改。

:::

:::warning

本指南只能帮助您尽可能接近原版。
不幸的是，目前在 Paper 中无法获得 100% 的原版体验。

:::

## 其他考虑因素

* `spigot.yml` 中的 `settings.timeout-time` 反映了 `server.properties` 文件中的看门狗设置。请记住，`spigot.yml` 中的设置以秒为单位，而默认设置以毫秒为单位。
* `paper-global.yml` 中的 `watchdog` 控制看门狗早期警告。虽然这只是一个警告，但您可能想要禁用它，因为它可能会变得烦人。
