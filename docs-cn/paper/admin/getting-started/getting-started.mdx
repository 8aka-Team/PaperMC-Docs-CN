# 入门指南

## 要求

:::tip

Paper 需要至少 **Java <Property name="PAPER_JAVA_MIN" />** 才能运行，[下载和安装很简单](/misc/java-install)。

:::

| Paper 版本     | 推荐的 Java 版本                        |
|----------------|-----------------------------------------|
| 1.8 到 1.11    | Java 8                                  |
| 1.12 到 1.16.4 | Java 11                                 |
| 1.16.5         | Java 16                                 |
| 1.17.1-1.18.1+ | Java <Property name="PAPER_JAVA_MIN" /> |

## 下载 Paper

Paper 直接从我们的[网站下载页面](https://papermc.io/downloads)提供可运行的服务器 JAR 文件。

点击构建版本号即可下载文件。

## 运行服务器

要运行服务器，您需要创建一个启动脚本或在终端中运行命令。

您可以使用我们的[启动脚本生成器](/misc/tools/start-script-gen)生成启动脚本。
您也可以在那里获取优化过的终端命令来运行服务器。

如果您只是想要一个简短的命令：
```bash
java -Xms4G -Xmx4G -jar paper.jar --nogui
```
确保您已将终端导航到服务器所在的目录，并且已将 `paper.jar` 替换为您下载的 JAR 文件的名称。

可以通过更改 `Xmx` 和 `Xms` 参数中的数字来设置 RAM 大小。
`--nogui` 禁用原版的 GUI，这样在使用命令行时就不会出现双重界面。

要配置您的服务器，请参阅[全局配置](../reference/configuration/global-configuration.mdx)和
[每个世界的配置](../reference/configuration/world-configuration.mdx)页面。

## 更新服务器

更新 Paper 很简单！查看我们的[更新教程](../how-to/update.md)了解更多信息。

## 迁移到 Paper

### 从原版迁移

从原版迁移很容易，但有一些区别，主要是在世界存档方面。Paper（以及
CraftBukkit 和 Spigot）将世界的每个维度（下界、末地等）分离到单独的
世界文件夹中。

Paper 会自动为您处理这个转换。无需额外考虑。

### 从 CraftBukkit 或 Spigot 迁移

Paper 可以直接替换 CraftBukkit 和 Spigot，您无需做任何更改。
