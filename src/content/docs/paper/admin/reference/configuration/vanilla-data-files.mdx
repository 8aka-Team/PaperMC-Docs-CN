## `eula.txt`

`eula.txt` 是一个简单的配置文件，允许你接受 [EULA](https://aka.ms/MinecraftEULA)，
运行 Minecraft 服务器需要接受该协议。

<ConfigNode data={parse(EulaYML)} lang="properties" path={["eula"]} />

## `banned-ips.json`

`banned-ips.json` 是一个 JSON 文件，用于存储不允许连接到服务器的 IP 地址。

:::caution[警告]

这是一个由服务器使用的数据文件，而不是一个配置文件。

我们不建议直接编辑此文件，它仅用于参考。

相反，应使用 `/ban-ip`、`/pardon-ip` 和 `/banlist ips` 命令来管理 `banned-ips.json` 文件。

:::

<ConfigNode data={parse(BannedIpsYML)} lang="json" path={["banned_ips"]} />


## `banned-players.json`

`banned-players.json` 是一个 JSON 文件，用于存储不允许连接到服务器的玩家。

:::caution[警告]

这是一个由服务器使用的数据文件，而不是一个配置文件。

我们不建议直接编辑此文件，它仅用于参考。

相反，应使用 `/ban`、`/pardon` 和 `/banlist players` 命令来管理 `banned-players.json` 文件。

:::

<ConfigNode data={parse(BannedPlayersYML)} lang="json" path={["banned_players"]} />

## `ops.json`

`ops.json` 是一个 JSON 文件，用于存储玩家的管理员权限信息。

在原版 Minecraft 中，它被用作一个简单的权限系统，并允许某些玩家绕过玩家数量限制。

:::caution[警告]

这是一个由服务器使用的数据文件，而不是一个配置文件。

我们不建议直接编辑此文件，它仅用于参考。

相反，应使用 `/op` 和 `/deop` 命令来管理 `ops.json` 文件。

:::

<ConfigNode data={parse(OpsYML)} lang="json" path={["ops"]} />

## `whitelist.json`

`whitelist.json` 是一个 JSON 文件，用于存储允许连接到服务器的玩家。

它与 `server.properties` 文件中的 [`white-list`](/paper/reference/server-properties#white_list) 选项配合使用。


:::caution[警告]

这是一个由服务器使用的数据文件，而不是一个配置文件。

我们不建议直接编辑此文件，它仅用于参考。

相反，应使用 `/whitelist` 命令来管理 `whitelist.json` 文件。

:::

<ConfigNode data={parse(WhitelistYML)} lang="json" path={["whitelist"]} />
