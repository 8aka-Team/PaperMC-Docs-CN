由于 Paper 中 Bukkit API 的实现方式，原版和 Paper 之间的游戏体验可能会存在不一致。
修复原版 Minecraft 中存在的漏洞也会进一步导致这种不一致。
尽管许多人可能不会注意到这些不一致，但在某些情况下它们可能会变得有问题。
一个例子是，技术型玩家可能会发现他们的机器无法像预期那样工作。
此页面旨在为希望尽可能接近原版的玩家提供一个起点。

:::note[注意]

此页面受 [Earthcomputer 的原版变更收集](https://gist.github.com/Earthcomputer/2296da33b8cc91dba81b48103c0e1fe3) 的启发。

:::

:::caution[警告]

本指南仅能帮助你尽可能接近原版游戏。
遗憾的是，目前在 Paper 中无法实现 100% 的原版体验。

:::

## 进一步考虑

* `spigot.yml` 中的 `settings.timeout-time` 与 `server.properties` 文件中的看门狗设置相对应。请注意，`spigot.yml` 中的设置单位是秒，而默认设置的单位是毫秒。
* `paper-global.yml` 中的 `watchdog` 控制看门狗的早期警告。虽然这只是一个警告，但你可能希望禁用它，因为它可能会让人感到烦扰。
